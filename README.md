# Global Assistant

Персональный Telegram-ассистент для развития: план дня из «сфер развития», трекинг навыков и прогресса, исследование идей и проактивные подсказки. Мультиагентная система на чистом Python — без тяжёлых фреймворков.

## Возможности

- **План дня** — текстом или голосом: «составь план на завтра: лаба по физике, урок aiogram, прогулка»
- **Голосовой ввод** — распознавание через faster-whisper локально
- **Трекинг задач** — кнопка «✅ Выполнено» прямо в сообщении с планом
- **Навыки** — ассистент помнит, что я умею и что изучаю
- **Нештрафующая дисциплина** — провален день → план автоматически пересобирается с меньшим объёмом; вместо «сгоревших стриков» — скользящее среднее за 2 недели; отдых — полноценная часть плана, а не награда

## Архитектура

```
Telegram (aiogram 3)
   │  текст / голос (faster-whisper)
   ▼
Оркестратор ── один LLM-вызов с tool calling, роутит запрос
   ├── Planner   — план дня, задачи, навыки          [готов]
   ├── Explorer  — поиск аналогов идей: arXiv,
   │               Semantic Scholar, GitHub Search    [фаза 2]
   ├── Muse      — проактивная генерация идей по крону [фаза 3]
   └── Coach     — ревью недели, адаптивная нагрузка   [фаза 3]
   ▼
SQLite (plans, tasks, skills, ideas, user_context, daily_stats)
```

### LLM: только бесплатные тарифы

Единая абстракция `LLMProvider` + роутер с цепочкой фолбэка: при 429 (квота) или 503 (перегрузка) запрос автоматически уходит к следующей модели.

```
gemini-3.1-flash-lite (500 RPD) → gemini-2.5-flash (20 RPD) → Groq llama-3.3-70b
```

Какая модель у какого агента — маппинг `AGENT_MODELS` в [config.py](config.py): «мозги» агента меняются одной строкой, что позволяет A/B-тестировать модели.

## Структура репозитория

```
agents/      оркестратор и суб-агенты (planner, explorer, muse, coach)
bot/         обработчики Telegram: текст, голос, кнопки
db/          схема SQLite и подключение
llm/         абстракция LLMProvider, провайдеры Gemini и Groq, роутер
prompts/     системные промпты агентов (markdown)
scheduler/   фоновые задачи APScheduler (фаза 3)
main.py      точка входа: сборка бота и polling
```

## Запуск

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # заполнить ключи
python3 main.py
```
